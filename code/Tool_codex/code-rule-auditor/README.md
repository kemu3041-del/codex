# 页面规范质检工具

这是一个浏览器本地运行的 React 页面规范检查工具，用于内部质检 HTML / CSS 是否尽量复用 `common.css` 里的基础规范。

## 使用方式

1. 直接打开 `index.html`。
2. 上传需要检查的项目文件夹，工具会自动筛选并罗列其中的 `.html`、`.htm`、`.css` 文件，忽略 `.min` 压缩插件文件。
3. 可选上传团队最新版 `common.css`，工具会提取里面的 class 和 CSS 属性值作为规范库。
4. 修改设计规则数值后会实时重新计算结果。
5. 在“问题清单”中点击“需处理 / 需确认”条目，可展开源码片段并高亮定位行。

## 当前内置规则

- 必需类名：默认检查 HTML 中是否出现 `title`、`subtitle`
- 工具类前缀：`fnt`、`mt`、`mb`、`pt`、`pb`
- 标题标签：`h1` 到 `h6`
- CSS 属性：`font-size`、`margin-top`、`margin-bottom`、`padding-top`、`padding-bottom`、`color`、`background-color`、`border-radius`
- 内置 common.css 值：`fnt12` 到 `fnt80`，`mt20` 到 `mt200` 等常用间距值
- 按钮样式：自动识别 `button` 标签和 `btn` / `button` 类，罗列按钮字号、颜色、内边距、圆角
- 问题定位：问题清单可展开 HTML / CSS 片段，标注疑似问题所在行

## 设计规则填写

设计师只需要填写数值，不需要懂选择器或 class：

- 必需类名，例如 `title,subtitle`
- 标题字号，例如 `36`
- 标题颜色，例如 `#333`
- 副标题字号，例如 `22`
- 副标题颜色，例如 `#999`
- 允许字号值，例如 `16,18,22,24,32,36,48`
- 允许颜色值，例如 `#333,#666,#999,#fff,#000`
- 允许间距值，例如 `20,30,60,80,100,120`
- 允许按钮圆角，例如 `4,6,8,10,999`
- 标题到副标题间距，例如 `30`

工具会把页面中实际出现的字号、颜色、margin / padding 间距、按钮字号、按钮颜色、按钮内边距和按钮圆角全部罗列出来，并标出“设计允许值”和“页面多出来的值”。

工具内部自动识别 `h1`、`h2` 和常见标题类名。开发同事如果要调整识别范围或新增规则，可在 `app.js` 顶部修改 `AUDIT_RULE_CONFIG`。

## 新增规则方式

规则集中写在 `AUDIT_RULE_CONFIG`：

- `selectors`：维护标题、副标题、按钮等识别范围。
- `designerFields`：维护设计师需要填写的字段。
- `designerRules`：维护每条质检规则使用哪个字段、检查哪个 CSS 属性。

以后新增规则优先只改这个对象，不需要为每条规则单独写检测函数。

## 后续可扩展方向

- 增加项目级规则模板，例如官网、专题页、活动页分别有不同检查阈值。
- 增加图片 alt、链接空 href、重复 id、按钮 hover 态等检查。
- 增加 CSV 导出，方便质检结果进入内部看板。
- 如果需要账号、历史记录、多人协作，再迁移到 React + 后端存储。

## 当前边界

工具优先做快速质检，不完整模拟浏览器最终计算样式。它会读取上传文件夹中的 CSS、HTML 内联 style，以及 `fnt36 / mt30 / mb30 / pt120 / pb120` 这类工具类映射出来的值。

项目使用本地 React UMD 文件：`vendor/react.production.min.js` 和 `vendor/react-dom.production.min.js`，不需要 `npm install`。
