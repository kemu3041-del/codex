# 设计图转 HTML/CSS/JS 结构工具

这是一个浏览器本地运行的设计图结构生成工具。上传 `jpg / png / webp` 后，工具会用 Canvas 读取图片尺寸、比例、色板、明暗和基础视觉密度，再生成三段可复制代码：

- `HTML`：页面结构、图片区域、标题、按钮、模块占位。
- `CSS`：根据图片色板生成 CSS 变量、响应式布局和基础组件样式。
- `JS`：入场显示、按钮事件绑定入口，方便后续接跳转、埋点或弹窗。

## 使用方式

1. 直接打开 `index.html`。
2. 上传一张完整设计图，例如 `design.jpg`。
3. 根据需要选择页面类型：
   - 自动判断
   - 落地页 / 官网首屏
   - SaaS / 数据看板
   - 移动端页面
   - CMS 页面模块
4. 查看右侧生成的 `HTML / CSS / JS`。
5. 复制当前代码，或点击“下载三文件”得到 `index.html`、`style.css`、`app.js`。

## 适合的交付场景

- 快速把设计图拆成可维护的前端骨架。
- 给 SaaS/CMS 源码区准备基础 HTML/CSS/JS。
- 从截图中提取色板并转成 CSS 变量。
- 做页面还原前的第一版结构搭建。

## CMS 字段建议

静态粘贴型页面优先直接替换 HTML 中的内容：

- 主标题：`h1`
- 副标题 / 描述：`.lead` 或段落文本
- 图片：`img src`
- 按钮文案：按钮文本
- 按钮链接：`href`
- 卡片列表：`article`

如果后续迁移到 React/Vite 工程，再把这些内容抽成 `cards`、`sections`、`products` 等数据对象。

## 当前边界

这个版本不调用云端 AI，也不会真正理解图片中的文字和复杂组件语义。它做的是本地启发式结构生成：尺寸、比例、色彩和页面类型建议。后续如果要做到“像素级还原”或“识别设计稿组件”，可以在这个工具上继续接入 OpenAI Vision、多模态模型或设计稿解析接口。
