# 磨砂弹窗截图还原经验

## 问题总结

第一版使用了多段渐变背景、渐变边框、`blur(14px)`、多层外发光、伪元素和 `mix-blend-mode`。技术上实现了完整的玻璃拟态，但视觉上偏向强发光的科幻 UI，没有准确匹配参考图中轻薄、克制的玻璃面板。

更准确的基础方案应由以下四部分组成：

1. 单一半透明蓝色背景。
2. 2px–4px 的轻微背景模糊。
3. 1px 半透明细边框。
4. `box-shadow` 组成的顶部内高光、底部反光、外轮廓和深色投影。

```css
.glass-panel {
  background: rgba(15, 50, 110, 0.4);
  backdrop-filter: blur(2px) saturate(1.5);
  -webkit-backdrop-filter: blur(2px) saturate(1.5);

  border: 1px solid rgba(160, 210, 255, 0.63);

  box-shadow:
    inset 0 1px 0 rgba(255, 255, 255, 0.32),
    inset 0 -1px 0 rgba(80, 140, 255, 0.13),
    0 0 0 1px rgba(80, 150, 255, 0.22),
    0 8px 32px rgba(0, 10, 40, 0.5);
}
```

## 两类方案的使用边界

### 准确还原型

使用半透明单色、轻模糊、普通细边框和多层阴影。适合参考图中透明度较高、边框较均匀、高光克制的弹窗。

### 强化效果型

使用渐变边框、强模糊、多层外发光、径向渐变和 `mix-blend-mode`。只适合参考图明确存在霓虹光晕、强烈明暗变化或局部颜色混合的情况。

## 后续实施规则

1. 先分析截图的背景透出程度、模糊程度、边框均匀度和高光位置。
2. 先交付最小准确方案，不提前加入截图中没有的技术效果。
3. 只在局部高光不能通过普通边框和内阴影完成时，才使用伪元素。
4. 只在高光需要与底图发生真实颜色混合时，才使用 `mix-blend-mode`和 `isolation` 。
5. 在包含纹理或光点的背景上验证 `backdrop-filter`；纯色背景不能有效展示磨砂差异。

## SaaS/CMS 复用

保留 `.glass-panel` 为独立样式块即可复用。标题、宽高、圆角、背景色和边框亮度可替换为 CMS 字段或 CSS 变量；不需要 JavaScript 或第三方依赖。
