# AGENTS.md

## 目录定位

`edu_codex/` 用于长期沉淀教学类可视化业务，不限定小学，也不只做数学。这里的项目可以覆盖 K12、素养教育、职业启蒙、跨学科项目、知识图谱、互动课件和可复制到 SaaS/CMS 平台的教学模块。

## 工作重点

- 每个教学项目都要先明确学习对象、知识目标、推理路径和可视化表达方式。
- 教学表达优先清晰、直观、可操作，不为了炫技牺牲理解成本。
- 复杂知识要拆成阶段：引入场景、观察现象、动手操作、规律总结、迁移练习。
- 交互设计要服务学习目标，例如拖拽、对比、分步演示、即时反馈、错误提示、结果验证。
- 视觉风格要适合教学业务：层级清楚、信息不拥挤、重点明确，避免纯装饰动画。
- 长期项目要沉淀成可复用组件、课程模板、知识点模板和 CMS 字段方案。

## 推荐目录结构

```text
edu_codex/
├── curriculum-map/       # 长期课程规划、知识图谱、年级/阶段映射
├── subjects/             # 学科分类沉淀
├── modules/              # 具体互动课件、可视化页面、业务 Demo
├── components/           # 可复用教学组件
├── lesson-templates/     # 课程脚本、页面模板、交互模板
├── research-notes/       # 参考案例、教学法、竞品拆解、灵感记录
└── assets/               # 图片、图标、音频、模型、公共素材
```

## 学科规划

- `subjects/math/`：数学推理、几何、数感、函数、统计、概率等。
- `subjects/chinese/`：语文阅读、字词句、作文结构、古诗文理解等。
- `subjects/english/`：单词、语法、听说读写、情景表达等。
- `subjects/science/`：物理、化学、生物、地理、实验模拟等。
- `subjects/arts/`：美术、音乐、设计启蒙、审美训练等。
- `subjects/technology/`：编程、AI、工程、信息素养等。
- `subjects/life-skills/`：财商、时间管理、安全教育、生活常识等。
- `subjects/interdisciplinary/`：跨学科项目式学习、综合实践任务等。

## 技术偏好

- 普通教学交互优先使用 HTML/CSS/JS，便于复制到 SaaS/CMS。
- 复杂状态、组件化课程或长期产品原型可以使用 React。
- 分步动画、滚动叙事、路径演示优先使用 GSAP。
- 几何、关系图、知识图谱可以使用 SVG、Canvas、D3 或 DOM。
- 3D 空间、实验场景、立体结构再考虑 Three.js。
- 静态 HTML 教学页面的主体文案、题目、选项、图片优先写在 HTML 中，动画和交互逻辑放在 JS 中。

## 每个教学项目必须沉淀

1. 教学目标：学生最终要理解什么。
2. 适用对象：年龄段、年级、基础要求。
3. 知识拆解：概念、公式、规则、误区和前置知识。
4. 推理路径：从现象到结论的分步过程。
5. 可视化方式：图形、动画、操作、对比或模拟。
6. 交互设计：学生如何操作，系统如何反馈。
7. 数据接入：题目、步骤、素材、评分、学习记录如何进入 CMS/SaaS。
8. 复用方式：如何沉淀为组件、模板或配置化模块。
9. 注意事项：移动端、可访问性、性能、素材版权和教学准确性。

## 交付说明必须包含

- 实现思路。
- 知识点和学习目标。
- 交互/动画拆解。
- 文件结构或关键文件说明。
- CMS/SaaS 字段接入建议。
- 验证方式和已知限制。
