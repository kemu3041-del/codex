# 06 阶段复盘

## 必须掌握

- 能解释 Next.js 和 React 的关系。
- 能看懂 `app/` 目录。
- 能创建 `page.js` 和 `layout.js`。
- 能引用静态资源。
- 能本地启动 Next.js 项目。

## 自查问题

- 我是否知道路由从哪里生成？
- 我是否理解 layout 为什么适合放公共结构？
- 我是否知道哪些资源应该放到 `public/`？
- 我是否能说明 Next.js 为什么适合内容和业务页面？

## 常见卡点

- 把 `page.js` 当普通组件随意放。
- 不理解 layout 会包裹子页面。
- 静态资源路径写错。

下一章深入 App Router，重点是动态路由、loading、error 和服务端/客户端组件边界。
