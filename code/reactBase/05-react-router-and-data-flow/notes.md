# 05 React 路由和数据流

## 学习目标

这一章学习多页面前端应用的组织方式：页面如何切换，数据如何在页面和组件之间流动。

## 核心概念

- 客户端路由：不刷新整个页面，只切换组件。
- 页面组件：负责路由对应的完整页面。
- 通用组件：负责复用的 UI。
- 路由参数：用 URL 表达当前资源。
- Context：跨层级共享少量全局状态。
- 异步状态：loading、error、empty、success。

## 和旧写法的对应关系

以前：

- 多个 HTML 文件通过链接跳转。
- 页面之间靠 URL、localStorage 或全局变量传递信息。
- 请求完成后手动插入 DOM。

现在：

- 路由负责页面切换。
- URL 参数表达当前详情页。
- 数据请求结果进入组件渲染流程。
- 所有异步状态都要在 UI 中体现。

## 数据流原则

- 页面级组件负责拿数据和组织布局。
- 展示组件只关心收到的数据。
- 不要为了方便到处写全局状态。
- loading、error、empty 是正式 UI，不是临时处理。

## 本章重点

- 能做列表页和详情页。
- 能通过路由参数找到对应数据。
- 能用 Context 管理主题或用户信息。
- 能完整处理请求状态。
