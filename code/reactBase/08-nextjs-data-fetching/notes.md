# 08 Next.js 数据请求

## 学习目标

这一章掌握 Next.js 中数据应该在哪里请求，以及不同渲染方式适合什么场景。

## 核心概念

- 服务端请求：在 Server Component 中直接请求数据。
- 客户端请求：在 Client Component 中根据用户交互请求数据。
- 静态渲染：适合不频繁变化的内容。
- 动态渲染：适合每次请求都可能变化的内容。
- 缓存：减少重复请求。
- 重新验证：让静态内容按规则更新。
- 环境变量：存放接口地址、密钥等配置。

## 选择方式

优先服务端请求：

- 首屏内容需要 SEO。
- 数据和用户交互无关。
- 不想把接口细节暴露给浏览器。

使用客户端请求：

- 搜索、筛选、分页强依赖用户操作。
- 数据需要频繁局部刷新。
- 使用浏览器状态或本地交互。

## 本章重点

- 不要所有请求都放到客户端。
- 不要把敏感配置暴露给浏览器。
- 请求状态要完整。
- 缓存策略要和业务更新频率一致。
