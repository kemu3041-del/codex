# 09 Next.js API 和轻量全栈

## 学习目标

这一章学习 Next.js 中的轻量后端能力，让你能在同一个项目里完成简单接口、表单提交和数据校验。

## 核心概念

- Route Handler：在 `app/api/` 下创建接口。
- GET：读取数据。
- POST：提交数据。
- Request：读取请求参数和请求体。
- Response：返回 JSON 或状态码。
- 服务端校验：不能只依赖前端校验。
- mock 数据：学习阶段先用本地数据模拟后端。

## 和纯前端写法的区别

以前：

- 表单提交只在浏览器中处理。
- 校验主要靠前端。
- 数据经常写死在页面里。

现在：

- 可以在 Next.js 项目中创建接口。
- 表单提交要处理服务端结果。
- 前后端需要约定输入输出结构。

## 接口设计基本要求

每个接口至少要明确：

- 请求方法。
- 请求路径。
- 输入字段。
- 成功返回。
- 失败返回。
- 错误状态码。

## 本章重点

- 表单提交必须处理 loading、success、error。
- 服务端要做基础字段校验。
- 客户端不要保存敏感逻辑。
- mock 数据可以帮助你先理解流程。
