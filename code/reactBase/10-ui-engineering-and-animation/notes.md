# 10 UI 工程化和动效集成

## 学习目标

这一章把你已有的页面还原、视觉动效、GSAP 和 Three.js 经验，接入 React/Next.js 的组件化工作流。

## 核心概念

- 组件化样式：样式跟组件职责对应。
- CSS Module：减少样式污染。
- 全局样式：适合 reset、变量和基础排版。
- GSAP 生命周期：初始化和清理要跟组件挂载卸载对应。
- Three.js 生命周期：场景、渲染器、事件和动画帧都要销毁。
- 响应式设计：组件尺寸不能只适配桌面。
- 性能降级：移动端和低性能设备要减少动画负担。

## 和静态动效写法的区别

以前：

- 一个 HTML 页面写结构。
- 一个 CSS 文件写全部样式。
- 一个 JS 文件初始化所有动画。

现在：

- 页面拆成组件。
- 样式按组件或模块管理。
- 动画逻辑放到对应组件，并在卸载时清理。
- 数据和配置可以从 props 或 CMS 字段进入。

## 动效组件基本流程

实现一个动效组件时，先确认：

- 动画作用在哪些元素上。
- 动画何时开始。
- 是否依赖滚动。
- 是否需要响应窗口变化。
- 组件销毁时是否要清理动画实例。

## 本章重点

- 不把所有动画写成全局脚本。
- 不让动画污染其他页面。
- 不忽略移动端性能。
- 视觉效果要服务业务信息层级。
