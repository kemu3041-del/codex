# 03 组件、Props 和 State

## 学习目标

这一章重点掌握 React 中最常见的数据流：父组件把数据传给子组件，子组件通过事件通知父组件，状态变化后 UI 更新。

## 核心概念

- props：父组件传给子组件的数据。
- state：组件内部可变化的数据。
- 状态提升：多个组件需要共享同一份状态时，把状态放到最近公共父组件。
- 受控组件：表单值由 React state 管理。
- 单向数据流：数据通常从父组件流向子组件。

## 和旧写法的对应关系

以前：

- 用全局变量保存页面状态。
- 输入框值从 DOM 中读取。
- 子区域直接改外层 DOM。

现在：

- 用 state 保存变化。
- 输入框的 `value` 绑定 state。
- 子组件通过回调函数通知父组件。

## 状态放在哪里

判断方式：

- 只有一个组件用：放在这个组件内部。
- 多个兄弟组件都用：放到它们共同的父组件。
- 很多层都用：后续再考虑 Context。

## 本章重点

- props 只读，不要在子组件中修改 props。
- state 更新要使用 set 函数。
- 表单输入建议使用受控组件。
- 组件通信要保持方向清晰。
