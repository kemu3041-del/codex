# 03 练习：待办事项和组件通信

## 练习 1：父传子

任务：

- 创建 `TodoItem`。
- 父组件传入待办标题、完成状态。
- 子组件负责展示。

验收标准：

- `TodoItem` 不写死内容。
- props 命名清晰。
- 已完成事项有不同样式。

## 练习 2：子传父

任务：

- 点击待办项的完成按钮。
- 子组件调用父组件传入的函数。
- 父组件更新对应待办状态。

验收标准：

- 子组件不直接修改父组件数据。
- 点击后 UI 正确更新。

## 练习 3：受控输入

任务：

- 做一个新增待办表单。
- 输入框值由 state 管理。
- 提交后新增到列表。

验收标准：

- 空内容不能提交。
- 提交后清空输入框。
- 新事项显示在列表中。

## 常见错误

- 直接修改数组，例如 `todos.push()` 后不触发更新。
- 子组件尝试修改 props。
- 输入框用了 `value` 但没有 `onChange`。

## 加餐挑战

增加“全部、未完成、已完成”三个筛选状态。
