# React + Next.js 学习路线图

这份路线图按从低到高的顺序设计，目标是让你从 HTML/CSS/JS 前端，逐步掌握 React、Next.js 和真实项目工作流。

## 01-js-to-react-thinking

学习目标：完成从 DOM 思维到组件思维的转换。

核心概念：

- React 为什么不直接操作 DOM。
- UI = state 的结果。
- JSX 和 HTML 的区别。
- 组件是可复用的 UI 函数。

必做练习：

- 把一个静态 HTML 卡片改写成 JSX。
- 把按钮点击改成 React 事件写法。
- 把手动 `querySelector` 操作改成状态驱动。

小项目产出：

- 一个“产品卡片 + 收藏按钮 + 状态变化”的 React 入门 Demo。

旧经验对应：

- HTML 结构对应 JSX。
- CSS 仍然负责样式。
- JS DOM 操作转换为 React state。

成熟工作流：

- 先拆 UI，再决定组件。
- 先想数据状态，再写交互。
- 不用 DOM 查询解决常规 UI 更新。

## 02-react-foundation

学习目标：掌握 React 项目的基本组成和运行方式。

核心概念：

- npm、package.json 和启动命令。
- React 入口文件。
- 组件导入导出。
- JSX 表达式、列表渲染、条件渲染。

必做练习：

- 创建多个组件并组合成页面。
- 用数组渲染卡片列表。
- 根据状态显示不同内容。

小项目产出：

- 一个“课程列表页”组件练习项目。

旧经验对应：

- 原来复制多段 HTML，现在用数组加组件渲染。
- 原来用 class 控制显示隐藏，现在用条件渲染。

成熟工作流：

- 组件按职责命名。
- 页面内容和组件结构分开思考。
- 不把所有代码堆在一个文件里。

## 03-react-components-and-state

学习目标：理解 props、state 和组件通信。

核心概念：

- props 是父组件传给子组件的数据。
- state 是组件自己的可变状态。
- 状态提升。
- 受控表单。

必做练习：

- 父组件向卡片组件传数据。
- 点击按钮修改当前组件状态。
- 子组件事件通知父组件。
- 做一个受控输入框。

小项目产出：

- 一个“待办事项 + 筛选 + 新增”的组件状态项目。

旧经验对应：

- 原来用全局变量保存状态，现在用 `useState`。
- 原来在 DOM 上读值，现在用受控组件保存输入值。

成熟工作流：

- 状态放在真正需要它的最近公共父组件。
- 子组件尽量通过 props 接收数据。
- 表单状态要清晰、可追踪。

## 04-react-hooks-workflow

学习目标：掌握 Hooks 组织页面逻辑的方式。

核心概念：

- `useState`。
- `useEffect`。
- `useRef`。
- 自定义 Hook。
- 副作用和清理函数。

必做练习：

- 做计数器和 tab 切换。
- 用 `useEffect` 模拟数据加载。
- 用 `useRef` 操作非渲染数据或 DOM 引用。
- 封装一个 `useToggle`。

小项目产出：

- 一个“搜索列表 + 加载状态 + 空状态”的 Hooks 项目。

旧经验对应：

- 原来在页面加载后统一绑定事件，现在逻辑跟组件生命周期绑定。
- 原来随意写全局状态，现在通过 Hook 管理局部逻辑。

成熟工作流：

- 副作用必须知道何时执行。
- 事件逻辑、数据逻辑和视图结构分清楚。
- 重复逻辑优先封装成自定义 Hook。

## 05-react-router-and-data-flow

学习目标：理解多页面前端应用和数据流组织。

核心概念：

- 客户端路由。
- 页面组件和通用组件。
- Context。
- 简单数据请求。
- 加载、错误、空数据状态。

必做练习：

- 创建列表页和详情页。
- 从列表进入详情。
- 用 Context 保存主题或用户信息。
- 给请求增加 loading 和 error 展示。

小项目产出：

- 一个“文章列表 + 文章详情 + 主题切换”的 React SPA。

旧经验对应：

- 原来多个 HTML 文件跳转，现在前端路由切换组件。
- 原来页面间靠 URL 或全局变量传值，现在用路由参数和共享状态。

成熟工作流：

- 页面级组件负责数据和布局。
- 通用组件负责展示和复用。
- 所有异步数据都要考虑 loading、error、empty。

## 06-nextjs-foundation

学习目标：理解 Next.js 为什么存在，以及它和普通 React 的区别。

核心概念：

- Next.js 项目结构。
- App Router。
- `app/page.js`。
- layout。
- 静态资源和样式。
- 构建和启动命令。

必做练习：

- 创建第一个 Next.js 页面。
- 创建全局 layout。
- 添加图片和 CSS。
- 对比 React SPA 和 Next.js 页面结构。

小项目产出：

- 一个“个人学习主页”的 Next.js 入门项目。

旧经验对应：

- 原来手动组织多个 HTML 文件，现在用文件系统路由。
- 原来所有内容都在浏览器执行，现在部分内容可在服务端生成。

成熟工作流：

- 先理解目录约定。
- 页面、布局、组件分层清楚。
- 每次开发都能本地启动和构建验证。

## 07-nextjs-app-router

学习目标：掌握 Next.js App Router 的页面组织能力。

核心概念：

- 路由目录。
- 嵌套 layout。
- 动态路由。
- not-found。
- loading。
- error。
- Server Component 和 Client Component。

必做练习：

- 创建多级页面。
- 创建动态详情页。
- 添加 loading 和 not-found。
- 把交互组件标记为 Client Component。

小项目产出：

- 一个“课程中心 + 课程详情”的 Next.js 路由项目。

旧经验对应：

- 原来靠文件名和链接组织页面，现在靠 app 目录自动生成路由。
- 原来所有 JS 都在前端，现在要区分服务端和客户端边界。

成熟工作流：

- 默认优先 Server Component。
- 有交互和浏览器 API 时才使用 Client Component。
- 路由状态和错误状态必须完整。

## 08-nextjs-data-fetching

学习目标：掌握 Next.js 中数据请求和渲染策略。

核心概念：

- 服务端数据请求。
- 客户端数据请求。
- 静态渲染。
- 动态渲染。
- 缓存和重新验证。
- 环境变量。

必做练习：

- 服务端请求文章列表。
- 客户端搜索或筛选。
- 使用环境变量配置接口地址。
- 对比静态和动态渲染效果。

小项目产出：

- 一个“内容资讯列表 + 详情 + 搜索”的数据项目。

旧经验对应：

- 原来用 `fetch` 后手动插 DOM，现在请求结果直接参与组件渲染。
- 原来只考虑浏览器请求，现在要考虑服务端请求和缓存。

成熟工作流：

- 根据页面更新频率选择渲染策略。
- 接口错误、空数据和加载状态必须设计。
- 敏感配置放环境变量。

## 09-nextjs-api-and-fullstack

学习目标：了解 Next.js 的轻量全栈能力。

核心概念：

- Route Handler。
- GET 和 POST。
- 表单提交。
- 服务端校验。
- mock 数据。
- 前后端边界。

必做练习：

- 创建一个 GET 接口。
- 创建一个 POST 表单提交接口。
- 做基础字段校验。
- 前端调用自己的接口。

小项目产出：

- 一个“留言表单 + 列表展示”的 Next.js 全栈小项目。

旧经验对应：

- 原来只写前端页面，现在可以在同一项目内写简单接口。
- 原来表单只是前端校验，现在要考虑服务端校验。

成熟工作流：

- 接口输入输出要清楚。
- 表单提交必须处理成功、失败和禁用状态。
- 不把敏感逻辑放在客户端。

## 10-ui-engineering-and-animation

学习目标：把你已有的视觉动效能力接入 React/Next.js 工作流。

核心概念：

- 组件化样式。
- CSS Module 或全局 CSS。
- GSAP 在 React 中的使用。
- Three.js 和 React 的边界。
- 响应式 UI。
- 动画销毁和性能。

必做练习：

- 做一个响应式 Hero。
- 做一个 GSAP 入场动画组件。
- 做一个滚动触发区块。
- 把静态动效改造成可复用组件。

小项目产出：

- 一个“产品展示页 + 入场动画 + 滚动动效”的视觉项目。

旧经验对应：

- 原来直接在全局脚本写动画，现在动画要跟组件挂载和销毁配合。
- 原来一个页面写到底，现在要拆成可维护组件。

成熟工作流：

- 动画初始化和清理必须成对出现。
- 样式不能污染整个项目。
- 移动端和低性能设备要有降级方案。

## 11-project-workflow-and-quality

学习目标：建立成熟 React/Next.js 项目工作流。

核心概念：

- 项目目录规范。
- Git 工作流。
- 代码格式化。
- lint。
- 构建验证。
- 调试方法。
- 组件复盘。

必做练习：

- 整理一个项目目录。
- 修复 lint 或构建问题。
- 写一份组件说明。
- 做一次项目复盘。

小项目产出：

- 一个“规范化 React/Next.js 项目模板”。

旧经验对应：

- 原来页面能跑就结束，现在要考虑可维护、可协作、可交付。
- 原来靠浏览器看效果，现在要加入构建和质量检查。

成熟工作流：

- 每次修改都能说明影响范围。
- 每个项目都能启动、构建、验证。
- 组件、数据、样式和副作用边界清楚。

## 12-real-world-saas-cms-project

学习目标：完成一个接近真实工作的 React/Next.js SaaS/CMS 项目。

核心概念：

- 需求拆解。
- 页面信息架构。
- CMS 字段映射。
- 组件配置化。
- 数据适配层。
- 交互和动效集成。
- 部署和交付说明。

必做练习：

- 拆解一个真实 SaaS/CMS 页面。
- 设计数据字段。
- 开发列表、详情、筛选和表单。
- 编写交付说明。

小项目产出：

- 一个“可配置内容展示平台”或“SaaS Dashboard 页面模板”。

旧经验对应：

- 原来把 HTML/CSS/JS 粘贴进平台，现在要能判断什么时候用静态交付，什么时候用工程化交付。
- 原来内容写死，现在要考虑 CMS 字段和数据适配。

成熟工作流：

- 需求先拆成页面、组件、数据、交互和交付方式。
- 代码要能被后续项目复用。
- 文档必须说明启动方式、字段映射、验证方式和已知限制。

## 总体验收标准

完成全部阶段后，你应该能独立完成：

- React 组件开发。
- Hooks 状态和副作用管理。
- 多页面 React 应用。
- Next.js App Router 项目。
- 服务端和客户端数据请求。
- 简单接口开发。
- SaaS/CMS 页面组件化和字段适配。
- 动效页面的 React/Next.js 集成。
- 项目启动、调试、构建、交付和复盘。
