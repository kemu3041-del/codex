# 12 真实 SaaS/CMS 项目训练

## 学习目标

这一章完成一个接近真实工作的 React/Next.js 项目，把需求拆解、组件开发、数据适配、动效集成和交付说明串起来。

## 核心概念

- 需求拆解：先确认页面目标和用户路径。
- 信息架构：确定页面模块和内容层级。
- 组件配置化：让标题、图片、按钮、列表可替换。
- CMS 字段映射：明确后台字段如何进入页面。
- 数据适配层：把接口数据转换成组件需要的结构。
- 交互状态：筛选、搜索、分页、弹窗、表单。
- 交付说明：启动、构建、字段、限制、验证。

## 项目建议方向

可以选择其中一个：

- SaaS Dashboard：指标卡、图表、列表、筛选、详情。
- CMS 内容站：栏目、文章列表、详情页、推荐内容。
- 产品展示页：Hero、功能区、案例、表单、动效。
- 教学课程页：课程列表、详情、学习进度、资源下载。

## 从旧经验升级

以前：

- 内容写死在 HTML。
- 动效写在全局 JS。
- 复制粘贴到平台后再改字段。

现在：

- 先定义字段。
- 再设计组件 props。
- 再写数据适配。
- 最后接 UI、交互和动效。

## 本章重点

- 不只做漂亮页面，要能交付和复用。
- 数据字段必须能解释。
- 组件要能适配不同内容。
- 动效不能破坏可维护性。
