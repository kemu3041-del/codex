# 12 练习：真实 SaaS/CMS 项目

## 练习 1：需求拆解

任务：

- 选择一个真实项目方向。
- 写出页面目标、核心用户、主要模块。
- 标出哪些内容来自 CMS。

验收标准：

- 页面目标明确。
- 模块拆分合理。
- CMS 字段初版清晰。

## 练习 2：字段设计

任务：

- 为页面设计字段。
- 区分内容字段和效果配置。
- 说明哪些字段必填。

验收标准：

- 字段命名语义化。
- 字段能支撑页面展示。
- 不把动效配置和内容数据混在一起。

## 练习 3：页面实现

任务：

- 开发完整页面。
- 实现核心交互。
- 接入模拟数据。

验收标准：

- 页面能启动。
- 核心模块完整。
- 移动端可用。
- 空状态和错误状态清楚。

## 练习 4：交付说明

任务：

- 编写项目 README。
- 说明启动、构建、字段映射、验证方式和已知限制。

验收标准：

- 其他人能按文档启动项目。
- CMS/SaaS 对接方式明确。
- 已知限制不隐藏。

## 加餐挑战

把项目改造成可复用模板，支持替换主题色、内容和模块开关。
