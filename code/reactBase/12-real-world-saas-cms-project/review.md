# 12 阶段复盘

## 必须掌握

- 能把需求拆成页面、组件、数据、交互和交付方式。
- 能设计 CMS 字段。
- 能开发完整 React/Next.js 页面。
- 能处理真实业务状态。
- 能写交付文档。

## 自查问题

- 页面是否解决明确业务目标？
- 字段设计是否能支撑 CMS 替换？
- 组件是否可复用？
- 是否验证了移动端？
- 是否说明了启动、构建和已知限制？

## 常见卡点

- 只做视觉，没有字段和数据设计。
- 组件写死，后续无法复用。
- 动效和业务逻辑耦合太深。
- 没有交付说明。

## 完成标准

完成本章后，你应该能独立接一个 React/Next.js 页面需求，并从需求拆解一直做到代码交付和复盘。
