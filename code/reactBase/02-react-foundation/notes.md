# 02 React 基础结构

## 学习目标

这一章开始理解一个 React 项目是怎么组织和运行的，不再只看单个组件片段。

## 核心概念

- `package.json`：记录依赖和启动命令。
- 入口文件：把 React 应用挂载到页面。
- 组件文件：一个组件负责一块 UI。
- import/export：拆分和复用组件。
- 列表渲染：用数组生成重复 UI。
- 条件渲染：用状态或数据决定显示内容。

## 和旧写法的对应关系

以前：

- 一个 HTML 文件写完整页面。
- 多个卡片靠复制 HTML。
- JS 文件统一处理全部交互。

现在：

- 一个页面由多个组件组合。
- 多个卡片由数组 `.map()` 渲染。
- 交互逻辑跟组件放在一起。

## 组件拆分原则

优先按职责拆：

- 页面组件：负责页面布局和数据组织。
- 展示组件：负责显示一块 UI。
- 交互组件：负责局部状态和事件。

不要为了拆而拆。一个组件如果只用一次，但逻辑复杂，也可以拆；一个组件如果重复出现，更应该拆。
