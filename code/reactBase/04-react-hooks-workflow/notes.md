# 04 React Hooks 工作流

## 学习目标

这一章学习如何用 Hooks 组织组件逻辑，重点是状态、副作用、引用和可复用逻辑。

## 核心概念

- `useState`：保存会触发 UI 更新的状态。
- `useEffect`：处理副作用，例如请求、订阅、计时器。
- `useRef`：保存不触发渲染的值，或拿到 DOM 引用。
- 自定义 Hook：把可复用逻辑抽出来。
- 清理函数：组件卸载或副作用重新执行前清理资源。

## 和旧写法的对应关系

以前：

- 页面加载后统一绑定事件。
- 定时器、请求、滚动监听散落在脚本中。
- 重复逻辑复制到多个页面。

现在：

- 副作用跟组件绑定。
- 组件销毁时清理副作用。
- 重复逻辑封装成自定义 Hook。

## useEffect 的基本判断

问自己三个问题：

- 这个逻辑是不是由渲染结果之外的事情引起？
- 它依赖哪些状态或 props？
- 它是否需要清理？

常见副作用：

- 数据请求。
- 事件监听。
- 定时器。
- 第三方动画库初始化。

## 本章重点

- 不要把所有逻辑都塞进 `useEffect`。
- 能在渲染中计算的内容，不要放进 state。
- 有监听、定时器、动画实例时，要考虑清理。
- 自定义 Hook 应该解决重复逻辑，不是为了显得复杂。
