# 11 项目工作流和质量

## 学习目标

这一章训练成熟开发者的工作方式：不只是写出页面，而是能让项目清晰、可维护、可验证、可交付。

## 核心概念

- 目录规范：按页面、组件、数据、工具函数区分职责。
- Git 工作流：每次修改能说明原因和范围。
- 格式化：减少无意义代码风格差异。
- lint：提前发现常见问题。
- 构建验证：确认代码能被生产环境打包。
- 调试：能定位组件、数据、样式和请求问题。
- 复盘：把问题沉淀成规则。

## 成熟工作方式

开始前：

- 明确目标。
- 拆页面结构。
- 判断数据来源。
- 判断交互和动效边界。

开发中：

- 小步实现。
- 随时运行验证。
- 发现问题及时记录。

完成后：

- 检查移动端。
- 检查空状态和错误状态。
- 检查构建。
- 写清楚交付说明。

## 本章重点

- 不把所有内容堆进一个文件。
- 不靠记忆维护项目结构。
- 每个功能都要能解释为什么这样拆。
- 每次交付都要有验证方式。
