# 11 练习：项目质量和交付流程

## 练习 1：整理项目目录

任务：

- 把一个页面项目整理成清晰目录。
- 区分 `components`、`data`、`lib`、`styles`。

验收标准：

- 文件职责清楚。
- 组件命名明确。
- 没有无意义的 `test1`、`new` 命名。

## 练习 2：构建验证

任务：

- 启动项目。
- 修复明显报错。
- 运行构建命令。

验收标准：

- 开发服务能启动。
- 构建能通过。
- 报错能说明原因。

## 练习 3：组件说明

任务：

- 为一个核心组件写说明。
- 包含用途、props、状态、交互和限制。

验收标准：

- 其他人能根据说明使用组件。
- 说明不写废话。
- 能指出移动端和数据限制。

## 加餐挑战

给一个项目写完整交付 README。
