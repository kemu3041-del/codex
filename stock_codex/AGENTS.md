# AGENTS.md

## 目录定位

`stock_codex/` 用于沉淀股票资讯、市场局势分析、技术指标、基本面研究、策略回测、数据看板和按天归档的分析总结。

## 重要边界

- 不输出确定性投资建议，不承诺收益。
- 分析必须说明数据来源、时间范围、市场假设和风险。
- 涉及最新行情、新闻、公告、财报、宏观数据、政策变化时，必须联网核验后再分析。
- 结论要区分事实、推断和个人策略假设。
- 对不完整数据要明确标注，不用模糊措辞伪装确定性。

## 工作重点

- 不定时获取最新资讯时，先按日期归档，再做摘要、影响路径和后续观察点。
- 当下局势分析要覆盖指数、行业、个股、宏观、资金面、情绪面和风险事件。
- 技术分析要记录指标参数，例如均线周期、成交量窗口、MACD/KDJ/RSI 参数。
- 基本面分析要记录财报周期、估值口径、同比/环比变化和可比公司。
- 策略研究要保存样本区间、买卖规则、仓位规则、回测限制和失败场景。

## 推荐目录结构

```text
stock_codex/
├── daily/             # 按天归档资讯、市场总结、观察清单
├── market-research/   # 行业、宏观、主题和公司研究
├── strategies/        # 策略规则、回测、复盘
├── data/              # 原始数据、清洗数据、数据说明
├── dashboards/        # 前端看板、图表 Demo
├── reports/           # 阶段性报告、周报、月报
└── templates/         # 日报、研究、策略模板
```

## 每日归档规则

- 每天的内容放在 `daily/YYYY/MM/YYYY-MM-DD.md`。
- 如果同一天有多次更新，可以在同一文件中按时间段追加。
- 每篇日报至少包含：市场概览、重要资讯、板块表现、个股观察、风险点、明日关注。
- 最新资讯必须保留来源链接、发布时间和抓取/核验时间。

## 技术偏好

- 数据处理、指标计算、回测可以使用 Python。
- 前端看板可以使用 HTML/CSS/JS、React、ECharts、D3 或 TradingView Lightweight Charts。
- 静态看板需要能直接打开或通过本地服务验证。
- 工程化看板需要说明启动命令、数据文件和字段映射。

## 每个股票项目必须沉淀

1. 分析目标：要回答什么市场或投资研究问题。
2. 数据来源：行情、公告、新闻、财报、宏观数据的来源和时间。
3. 分析框架：指数、行业、个股、资金、情绪、政策或基本面。
4. 指标逻辑：公式、参数、窗口、适用条件。
5. 结论分层：事实、推断、策略假设、风险。
6. 可视化方式：图表、表格、看板、时间线。
7. 复用方式：日报模板、策略模板、数据脚本或看板组件。
8. 验证方式：数据核验、图表检查、回测结果和已知限制。

## 交付说明必须包含

- 分析目标。
- 数据来源和核验时间。
- 关键结论与风险提示。
- 文件结构或关键文件说明。
- 图表/脚本运行方式。
- 后续观察点。
